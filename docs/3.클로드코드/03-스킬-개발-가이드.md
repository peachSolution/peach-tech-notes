## 1. 빠른 시작

> ⭐ **여기서 시작하세요** - 신규 모듈 개발 가이드

### 1.1 사전 체크리스트

```
시작 전 확인:
□ DB 스키마 파일 존재 (api/db/schema/[도메인]/[테이블].sql)
□ test-data 가이드 코드 확인 (api/src/modules/test-data/)
□ 개발 서버 실행 가능 상태
```

### 1.2 명령어 순서

```
┌─────────────────────────────────────────────────────────────────┐
│  신규 모듈 개발 (예: notice_board)                               │
│                                                                 │
│  1️⃣ Backend (15-20분)                                          │
│     /gen-backend notice_board file=Y excel=Y                    │
│                                                                 │
│     ✅ 종료 조건:                                               │
│     □ bun test src/modules/notice_board/ 통과                   │
│     □ bun run lint:fixed 성공                                   │
│     □ bun run build 성공                                        │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  2️⃣ Store (10-15분)                                            │
│     /gen-store notice_board file=Y excel=Y                      │
│                                                                 │
│     ✅ 종료 조건:                                               │
│     □ npx vitest src/modules/notice_board/ 통과                 │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  3️⃣ UI (20-30분)                                               │
│     /gen-ui notice_board ui=crud file=Y excel=Y                 │
│                                                                 │
│     ✅ 종료 조건:                                               │
│     □ npx vue-tsc --noEmit 성공                                 │
│     □ bun run lint:fix 성공                                     │
│     □ bun run build 성공                                        │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  4️⃣ 최종 검증                                                   │
│     webapp-testing 스킬로 브라우저 자동 테스트                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 옵션 선택 가이드

```
┌─────────────────────────────────────────────────────────────────┐
│  옵션 선택 기준                                                  │
│                                                                 │
│  file=Y 선택 시:                                                │
│  • 첨부파일 업로드/다운로드 필요                                │
│  • 이미지 업로드 필요                                           │
│  • 예: 공지사항 첨부파일, 상품 이미지                           │
│                                                                 │
│  excel=Y 선택 시:                                               │
│  • 엑셀 파일로 대량 데이터 업로드 필요                          │
│  • 목록 데이터 엑셀 다운로드 필요                               │
│  • 예: 회원 일괄등록, 상품 목록 다운로드                        │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ui 옵션 (기본 패턴):                                           │
│  • crud            → 목록 + 모달 방식 (기본, 가장 많이 사용)    │
│  • page            → 목록 + 별도 페이지 전환                    │
│  • two-depth       → 좌우 분할 레이아웃                         │
│  • infinite-scroll → 무한 스크롤                                │
│  • select-list     → 선택 모달 (다중 선택)                      │
│  • show-more       → 더보기 버튼 방식                           │
│                                                                 │
│  ui 옵션 (고급 패턴 - MCP 활용):                                │
│  • adv-search      → 복합 검색 (5개 이상 조건)                  │
│  • batch-process   → 순차 처리 (일괄 작업)                      │
│  • calendar        → 달력 UI                                    │
│  • kanban          → 칸반 보드                                  │
│  • mega-form       → 대량 입력 폼 (50개+)                       │
│  • tab-list        → 탭 내 리스트                               │
│                                                                 │
│  ⚠️ UI 패턴은 스킬 실행 시 대화형으로 선택합니다                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 핵심 원칙

### 2.1 왜 순차 개발인가?

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ❌ 병렬 생성의 문제                                             │
│  ─────────────────────                                          │
│  Backend + Store + UI 동시 개발                                 │
│      ↓                                                          │
│  각 레이어가 "추정"으로 개발                                    │
│      ↓                                                          │
│  API 스펙 불일치 발생                                           │
│      ↓                                                          │
│  수정 반복 → 실제로 더 느림                                     │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ✅ 순차 생성의 장점                                             │
│  ─────────────────────                                          │
│  Backend 완료 (TDD 통과) → 확정된 API 스펙                      │
│      ↓                                                          │
│  Store 완료 (TDD 통과) → 확정된 인터페이스                      │
│      ↓                                                          │
│  UI 개발 → 확정된 스펙 기반                                     │
│      ↓                                                          │
│  수정 최소화 → 실제로 더 빠름                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 개발자 역할 전환

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   기존: 코더                     신규: 조율자                    │
│   ─────────────                  ─────────────                   │
│   코드 직접 작성          →      의도 명시 (스킬 호출)           │
│   직접 테스트             →      TDD 통과 확인                   │
│   모든 결과 검증          →      최종 동작만 확인                │
│                                                                 │
│   핵심: "코드 작성" → "AI와 티키타카로 품질 확보"               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 핵심 공식

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│      전문 스킬 + AI 티키타카 + TDD = 완성도 높은 코드           │
│                                                                 │
│   gen-backend / gen-store / gen-ui                              │
│       ↓                                                         │
│   단계별 생성 + TDD 검증                                        │
│       ↓                                                         │
│   AI와 티키타카 (에러 수정 반복)                                │
│       ↓                                                         │
│   완성도 높은 코드                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 스킬 체계

### 3.1 저장소 구조

```
[프로젝트]/.claude/
├── skills/
│   ├── gen-prd/                   ← PRD 생성
│   ├── gen-db/                    ← DB DDL 생성
│   ├── gen-backend/               ← Backend API ⭐
│   │   ├── SKILL.md
│   │   └── references/            ← 상세 패턴 가이드
│   ├── gen-store/                 ← Frontend Store ⭐
│   │   ├── SKILL.md
│   │   └── references/
│   ├── gen-ui/                    ← Frontend UI ⭐
│   │   ├── SKILL.md
│   │   └── references/
│   ├── add-api/                   ← 외부 API 연동
│   ├── add-cron/                  ← Cron 작업
│   └── add-print/                 ← 인쇄 페이지
└── commands/
```

### 3.2 스킬 목록

#### 메인 워크플로우 (순서대로)

| 순서 | 스킬 | 커맨드 | 설명 | TDD |
|:----:|------|--------|------|:---:|
| 1 | gen-prd | `/gen-prd` | 대화형 PRD 문서 생성 | - |
| 2 | gen-db | `/gen-db` | DB DDL 마이그레이션 생성 | - |
| 3 | **gen-backend** | `/gen-backend` | Backend API 전문 생성 | ✅ |
| 4 | **gen-store** | `/gen-store` | Frontend Store 전문 생성 | ✅ |
| 5 | **gen-ui** | `/gen-ui` | Frontend UI 전문 생성 | - |

#### 추가 기능

| 스킬 | 커맨드 | 설명 |
|------|--------|------|
| add-api | `/add-api` | 외부 REST API 연동 |
| add-cron | `/add-cron` | Cron 스케줄러 |
| add-print | `/add-print` | 인쇄 전용 페이지 |

## 4. 스킬 상세

### 4.1 gen-backend

```
┌─────────────────────────────────────────────────────────────────┐
│  /gen-backend [모듈명] file=Y/N excel=Y/N                       │
│                                                                 │
│  전제조건: DB 스키마 파일 존재                                  │
│                                                                 │
│  생성 파일:                                                     │
│  ├── type/[모듈명].type.ts      ← Entity, DTO                  │
│  ├── dao/[모듈명].dao.ts        ← 데이터 접근                  │
│  ├── service/[모듈명].service.ts ← 비즈니스 로직               │
│  ├── service/[모듈명]-tdd.service.ts ← TDD 헬퍼                │
│  ├── controller/[모듈명].validator.ts ← 입력 검증              │
│  ├── controller/[모듈명].controller.ts ← API 엔드포인트        │
│  └── test/[모듈명].test.ts      ← TDD 테스트                   │
│                                                                 │
│  API 엔드포인트:                                                │
│  • GET    /[모듈명]/paging       - 페이징 목록                  │
│  • GET    /[모듈명]/list         - 전체 목록                    │
│  • GET    /[모듈명]/:seq         - 상세 조회                    │
│  • POST   /[모듈명]              - 등록                         │
│  • PUT    /[모듈명]/:seq         - 수정                         │
│  • DELETE /[모듈명]/:seq         - 삭제                         │
│  • PATCH  /[모듈명]/:seq/use     - 활성화/비활성화              │
│  • POST   /[모듈명]/excel/upload - 엑셀 업로드 (excel=Y)        │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ✅ 종료 조건:                                                   │
│  □ bun test src/modules/[모듈명]/ 통과                          │
│  □ bun run lint:fixed 성공                                      │
│  □ bun run build 성공                                           │
│                                                                 │
│  ⚠️ 자주 나는 오류:                                              │
│  • "Cannot find module" → import 경로 확인                      │
│  • "Type mismatch" → Entity/DTO 타입 정의 확인                  │
│  • "Query error" → DAO SQL 문법 확인 (PostgreSQL/MySQL 차이)    │
│                                                                 │
│  → 다음 단계: /gen-store                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 gen-store

```
┌─────────────────────────────────────────────────────────────────┐
│  /gen-store [모듈명] file=Y/N excel=Y/N                         │
│                                                                 │
│  전제조건: Backend API 완료 (TDD 통과)                          │
│                                                                 │
│  생성 파일:                                                     │
│  ├── type/[모듈명].type.ts      ← Frontend 타입                │
│  ├── store/[모듈명].store.ts    ← Pinia Store                  │
│  └── test/[모듈명].test.ts      ← TDD 테스트                   │
│                                                                 │
│  Store Actions:                                                 │
│  ├── paging / list / cursorList ← 목록 조회                    │
│  ├── detail / detailDataInit    ← 상세 / 초기화                │
│  ├── insert / update            ← 등록 / 수정                  │
│  ├── updateUse / softDelete     ← 활성화 / 삭제                │
│  ├── uploadFileLocal / uploadFileS3 ← 파일 업로드 (file=Y)     │
│  └── downloadExcel / excelUpload ← 엑셀 처리 (excel=Y)         │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ✅ 종료 조건:                                                   │
│  □ npx vitest src/modules/[모듈명]/ 통과                        │
│                                                                 │
│  ⚠️ 자주 나는 오류:                                              │
│  • "API 404" → Backend 서버 실행 여부 확인                      │
│  • "Type error" → Backend Entity와 Frontend Type 일치 확인      │
│  • "Auth error" → 테스트 시 인증 토큰 확인                      │
│                                                                 │
│  → 다음 단계: /gen-ui                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 gen-ui

```
┌─────────────────────────────────────────────────────────────────┐
│  /gen-ui [모듈명] file=Y/N excel=Y/N                            │
│                                                                 │
│  ⚠️ UI 패턴은 스킬 실행 시 대화형으로 선택 (필수)                │
│                                                                 │
│  전제조건: Store 존재 (Mock 데이터 또는 API 연결 완료)          │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  기본 UI 패턴:                                                  │
│  │ 패턴             │ 설명                    │ 참조 경로       │
│  ├──────────────────┼─────────────────────────┼─────────────────│
│  │ crud             │ 목록 + 모달 (기본)      │ pages/crud/     │
│  │ page             │ 목록 + 별도 페이지 전환 │ detail-page.vue │
│  │ two-depth        │ 좌우 분할 레이아웃      │ pages/two-depth/│
│  │ infinite-scroll  │ 무한 스크롤             │ infinite-scroll/│
│  │ select-list      │ 선택 모달 (다중)        │ select-list/    │
│  │ show-more        │ 더보기 버튼 방식        │ show-more-list/ │
│                                                                 │
│  고급 UI 패턴 (MCP 활용):                                       │
│  │ 패턴             │ 설명                    │ 필요 MCP        │
│  ├──────────────────┼─────────────────────────┼─────────────────│
│  │ adv-search       │ 복합 검색 (5개+ 조건)   │ Context7        │
│  │ batch-process    │ 순차 처리 (일괄 작업)   │ Context7        │
│  │ calendar         │ 달력 UI                 │ Context7+Seq    │
│  │ kanban           │ 칸반 보드               │ Context7+Seq    │
│  │ mega-form        │ 대량 입력 폼 (50개+)    │ Context7        │
│  │ tab-list         │ 탭 내 리스트            │ Context7        │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  생성 파일 (crud 패턴 예시):                                    │
│  ├── pages/[페이지명]/                                          │

## 8. 프로젝트별 적용

> 이 시스템은 **피치솔루션 기반 모든 프로젝트**(backoffice, www 및 향후 추가 프로젝트)에 동일한 구조로 적용됩니다.

### 8.1 적용 프로젝트

| 프로젝트 | 설명 | 상태 |
|----------|------|------|
| **backoffice** | 관리자 화면 | 적용 완료 |
| **www** | 일반 이용자 화면 | 적용 완료 |
| **[신규]** | 향후 추가 프로젝트 | 동일 구조 적용 |

### 8.2 공통 구성 원칙

| 항목 | 내용 |
|------|------|
| 디렉토리 구조 | `api/` + `front/` 동일 |
| Skills | 동일한 스킬 세트 (gen-backend, gen-store, gen-ui 등) |
| Commands | 동일한 커맨드 세트 (/gen-backend, /gen-store, /gen-ui 등) |
| Sub-agents | 동일한 서브에이전트 구성 |
| 가이드 코드 | `test-data/` 패턴 동일 |
| 순차 전략 | Backend → Store → UI |

### 8.3 프로젝트 구조

```
[프로젝트]/
├── .claude/
│   ├── skills/                ← 스킬 정의
│   │   ├── gen-backend/
│   │   ├── gen-store/
│   │   ├── gen-ui/
│   │   └── ...
│   ├── commands/              ← 커맨드 정의
│   └── sub-agents/            ← 서브에이전트 정의
├── api/                       ← Backend
│   └── src/modules/test-data/ ← 가이드 코드
├── front/                     ← Frontend
│   └── src/modules/test-data/ ← 가이드 코드
└── docs/
    ├── prompts/PROMPTS.md     ← 프롬프트 모음
    └── workflow/plans/        ← PRD 저장소
```

### 8.4 신규 프로젝트 적용 방법

```
┌─────────────────────────────────────────────────────────────────┐
│  신규 프로젝트 세팅 (3단계)                                      │
│                                                                 │

│     cp -r [기존]/.claude/ [새]/.claude/                         │
│                                                                 │
│  2️⃣ test-data 가이드 코드 복사                                  │

│                                                                 │

│                                                                 │
│  💡 복사 후 체크리스트:                                          │
