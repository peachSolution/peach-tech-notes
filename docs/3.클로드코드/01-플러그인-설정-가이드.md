## 1. 플러그인 시스템 구조

Claude Code 플러그인 시스템은 세 계층으로 구성됩니다:

```
마켓플레이스 (Marketplace)
  └── 플러그인 (Plugin)
        ├── 스킬 (Skills) — /slash-command 형태
        ├── LSP 서버 (Language Server Protocol)
        ├── MCP 서버 (Model Context Protocol)
        └── Hooks / Commands
```

- **마켓플레이스**: GitHub 레포 기반. `extraKnownMarketplaces`로 등록
- **플러그인**: 마켓플레이스 내 개별 단위. `enabledPlugins`로 활성화
- **스킬**: 플러그인이 제공하는 `/<스킬명>` 형태의 Claude 지시 파일

### /plugin 명령어 요약

| 명령어 | 설명 |
|--------|------|
| `/plugin` | 마켓플레이스 브라우징 및 플러그인 탐색 |
| `/plugin marketplace add <owner/repo>` | 새 마켓플레이스 추가 |
| `/plugin install <플러그인@마켓>` | 플러그인 설치 (`enabledPlugins`에 추가) |
| `/plugin uninstall <플러그인@마켓>` | 플러그인 제거 |

## 2. 문서 스킬 설치 (skills.sh)

### 왜 skills.sh를 사용하는가?

`anthropics/skills` 레포의 `document-skills` 플러그인은 `strict: false` 설정으로 인해
pdf, docx, pptx, xlsx **4개만 원해도 17개 전체가 로드**됩니다.
이는 Claude Code의 알려진 버그([#13344](https://github.com/anthropics/claude-code/issues/13344))로,
`skills` 배열이 무시되고 디렉토리 전체가 재귀 탐색됩니다.

**skills.sh**([Vercel Labs](https://github.com/vercel-labs/skills))를 사용하면
`--skill` 플래그로 **개별 스킬 단위 설치**가 가능하여 이 문제를 완전히 우회합니다.

### 설치 명령어

```bash
# 4개 문서 스킬을 글로벌 설치 (Claude Code, Codex 등 모든 에이전트)
npx skills add anthropics/skills -g -y --skill pdf --skill docx --skill pptx --skill xlsx
```

### 설치 구조

```
~/.agents/skills/          ← 원본 스킬 저장소 (skills.sh 관리)
  ├── pdf/
  ├── docx/
  ├── pptx/
  └── xlsx/

~/.claude/skills/          ← Claude Code 글로벌 스킬 (심볼릭 링크)
  ├── pdf → ~/.agents/skills/pdf
  ├── docx → ~/.agents/skills/docx
  ├── pptx → ~/.agents/skills/pptx
  └── xlsx → ~/.agents/skills/xlsx

~/.codex/skills/           ← Codex CLI 스킬 (심볼릭 링크)
  ├── pdf → ~/.agents/skills/pdf
  ├── docx → ~/.agents/skills/docx
  ├── pptx → ~/.agents/skills/pptx
  └── xlsx → ~/.agents/skills/xlsx
```

> **참고**: Cursor는 Claude Code의 `~/.claude/skills/`를 자동으로 인식하므로 별도 설치 불필요.
> Codex CLI는 skills.sh가 "universal" 방식으로 `~/.agents/skills/`에 설치하지만
> `~/.codex/skills/`에 심볼릭 링크가 자동 생성되지 않아 수동 링크 필요.

### 관리 명령어

| 명령어 | 설명 |
|--------|------|
| `npx skills list` | 설치된 스킬 목록 |
| `npx skills add <repo> --list` | 설치 가능한 스킬 목록 확인 |
| `npx skills check` | 업데이트 확인 |
| `npx skills update` | 스킬 업데이트 |
| `npx skills remove <skills>` | 스킬 제거 |
| `npx skills find <query>` | 스킬 검색 |

### 주요 플래그

| 플래그 | 설명 |
|--------|------|
| `-g, --global` | 글로벌 설치 (`~/.claude/skills/`) |
| `-s, --skill <name>` | 특정 스킬만 설치 (복수 지정 가능) |
| `-y, --yes` | 확인 프롬프트 스킵 |
| `-l, --list` | 설치하지 않고 목록만 확인 |
| `-a, --agent <name>` | 특정 에이전트에만 설치 |

## 3. claude-plugins-official에서 개별 스킬 설치

`claude-plugins-official` 마켓플레이스는 플러그인별로 `source`가 분리되어 있어
`strict` 문제 없이 원하는 플러그인만 선택 설치할 수 있습니다.

### 마켓플레이스 추가

```bash
/plugin marketplace add anthropics/claude-plugins-official
```

### 스킬 설치

```bash
# frontend-design 스킬 설치
/plugin install frontend-design@claude-plugins-official

# skill-creator 스킬 설치
/plugin install skill-creator@claude-plugins-official
```

설치 후 `/frontend-design`, `/skill-creator` 명령어로 즉시 사용 가능.

### claude-plugins-official 주요 카테고리

| 카테고리 | 대표 플러그인 |
|---------|------------|
| LSP (언어 서버) | typescript-lsp, pyright-lsp, gopls-lsp, rust-analyzer-lsp, lua-lsp 등 |
| 개발 도구 | agent-sdk-dev, feature-dev, plugin-dev, code-simplifier, frontend-design, skill-creator |
| 생산성 | pr-review-toolkit, commit-commands, code-review, hookify, claude-code-setup |
| 외부 서비스 | github, gitlab, supabase, slack, vercel, stripe, firebase, figma 등 |
| 보안 | security-guidance, semgrep, sonatype-guide |
| 학습 | explanatory-output-style, learning-output-style |

## 4. 설정 파일 레퍼런스

### 디렉토리 레이아웃

```
~/.claude/
├── settings.json                    # 전역 설정 (enabledPlugins, extraKnownMarketplaces 등)
├── settings.local.json              # 프로젝트별 로컬 설정 (git 비추적)
├── plugins/
│   ├── blocklist.json               # 차단 플러그인 목록
│   └── marketplaces/
│       ├── claude-plugins-official/ # 공식 플러그인 마켓 클론
│       │   └── .claude-plugin/
│       │       └── marketplace.json
│       └── peach-harness/           # PeachSolution 전용 마켓 클론
│           └── .claude-plugin/
│               └── marketplace.json
├── skills/                          # 글로벌 스킬 (skills.sh 심볼릭 링크 포함)
│   ├── pdf → ~/.agents/skills/pdf
│   ├── docx → ~/.agents/skills/docx
│   └── ...
│
~/.agents/skills/                    # skills.sh 원본 스킬 저장소
│   ├── pdf/
│   ├── docx/
│   └── ...
│
~/.codex/skills/                     # Codex CLI 스킬 (수동 심볼릭 링크)
```

### marketplace.json 플러그인 정의 스펙

```json
{
  "name": "플러그인명",
  "description": "설명",
  "source": "./plugins/플러그인명",   // 또는 "./" (레포 루트)
  "strict": true,                    // true: skills 배열만, false: 자동 탐지
  "skills": ["./skills/스킬명"],     // strict: true일 때 명시적 스킬 목록
  "lspServers": { ... },             // LSP 서버 정의 (LSP 플러그인)
  "category": "development",
  "author": { "name": "...", "email": "..." },
  "homepage": "https://..."
}
```

### blocklist.json 구조

```json
{
  "fetchedAt": "ISO 날짜",
  "plugins": [
    {
      "plugin": "플러그인명@마켓플레이스ID",
      "added_at": "ISO 날짜",
      "reason": "이유 코드",
      "text": "상세 설명"
    }
  ]
}
```

## 6. 주의사항

### anthropics/skills의 strict:false 문제

`anthropics/skills` 레포의 `marketplace.json`에서 `document-skills` 플러그인:

```json
{
  "name": "document-skills",
  "source": "./",           // 레포 루트 전체
  "strict": false,          // 스킬 목록 외 자동 발견 허용
  "skills": ["./skills/xlsx", "./skills/docx", "./skills/pptx", "./skills/pdf"]
}
```

- `strict: false` + `source: "./"` → 17개 스킬 전체 로드
- `strict: true`로 변경해도 Bug [#13344](https://github.com/anthropics/claude-code/issues/13344)로 `skills` 배열 무시됨
- blocklist는 플러그인 단위 차단만 가능 → 네임스페이스 우회로 무효화

**해결**: skills.sh로 개별 스킬을 `~/.claude/skills/`에 설치하고, `document-skills` 플러그인 비활성화.

### 관련 GitHub Issues

| Issue | 내용 | 상태 |
|-------|------|------|
| [#13344](https://github.com/anthropics/claude-code/issues/13344) | Plugin enable/disable 무시 — 모든 스킬 로드 | Open |
| [#14920](https://github.com/anthropics/claude-code/issues/14920) | 개별 플러그인 스킬 비활성화 기능 요청 | Open |
| [#22345](https://github.com/anthropics/claude-code/issues/22345) | Plugin 스킬에서 `disable-model-invocation` 미지원 | Open |

### Codex CLI 수동 심볼릭 링크

skills.sh가 Codex를 "universal" 방식으로 `~/.agents/skills/`에 설치하지만,
`~/.codex/skills/`에 심볼릭 링크가 자동 생성되지 않습니다. 수동 생성 필요:

```bash
ln -s ~/.agents/skills/pdf ~/.codex/skills/pdf
ln -s ~/.agents/skills/docx ~/.codex/skills/docx
ln -s ~/.agents/skills/pptx ~/.codex/skills/pptx
ln -s ~/.agents/skills/xlsx ~/.codex/skills/xlsx
```

### autoUpdate와 로컬 수정 충돌

- `autoUpdate: true` 마켓플레이스: Claude Code 실행 시 자동 git pull
- 로컬 `marketplace.json` 수정 시 autoUpdate로 인해 초기화 위험
- skills.sh 방식은 마켓플레이스와 독립적이므로 이 문제 없음
