## 실제 사용 예시

### COMMAND 사용 예시

```
┌─────────────────────────────────────────────────────────────────┐
│  💻 COMMAND: 정해진 워크플로우를 정확한 명령으로 실행           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  사용자: /sc:analyze src/auth --focus security                  │
│                    ↓                                            │
│  Claude: [analyze.md 프롬프트 로드]                             │
│          [보안 분석 워크플로우 실행]                             │
│          [결과 리포트 생성]                                      │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  사용자: /my:commit                                             │
│                    ↓                                            │
│  Claude: [commit.md 프롬프트 로드]                              │
│          [git diff 분석]                                        │
│          [한글 커밋 메시지 생성]                                 │
│          [커밋 실행]                                             │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  📁 파일 위치: ~/.claude/commands/sc/analyze.md                 │
│               ~/.claude/commands/my/commit.md                   │
└─────────────────────────────────────────────────────────────────┘
```

## 파일 구조 비교

```
~/.claude/
├── commands/                    # COMMAND
│   ├── sc/
│   │   ├── analyze.md          # /sc:analyze
│   │   ├── test.md             # /sc:test
│   │   └── implement.md        # /sc:implement
│   └── my/
│       ├── commit.md           # /my:commit
│       └── review.md           # /my:review
│
├── skills/                      # SKILL
│   ├── gen:backend-crud/
│   │   ├── SKILL.md            # 스킬 정의
│   │   └── RULES.md            # 프로젝트 규칙
│   ├── gen:frontend-ui/
│   │   ├── SKILL.md
│   │   └── RULES.md
│   └── comm:email/
│       └── SKILL.md
│
└── agents/                      # SUB AGENT
    ├── code-reviewer.md        # 코드 리뷰 agent
    ├── test-runner.md          # 테스트 실행 agent
    ├── backend-architect.md    # 백엔드 설계 agent
    └── git-commit-helper.md    # Git 커밋 agent
```

## 핵심 차이점 요약

```
┌───────────────────────────────────────────────────────────────────┐
│  🎯 가장 중요한 차이점                                            │
├───────────────────────────────────────────────────────────────────┤
│                                                                   │
│  COMMAND    →  "/명령어"로 정확히 호출해야 함                      │
│              →  프롬프트 확장 (동일 컨텍스트)                       │
│                                                                   │
│  SKILL      →  정확한 스킬 이름으로 호출해야 함                    │
│              →  코드 생성 + 규칙 적용 (동일 컨텍스트)               │
│                                                                   │
│  SUB AGENT  →  "자연어"로 자동 위임됨  ⭐                          │
│              →  독립 컨텍스트 + 도구/모델 제한 가능                 │
│              →  세션 간 재개 가능                                  │
│                                                                   │
└───────────────────────────────────────────────────────────────────┘
```
