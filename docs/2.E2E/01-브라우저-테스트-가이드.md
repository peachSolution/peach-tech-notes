# 브라우저 UI 테스트 가이드

Chrome DevTools MCP를 활용한 브라우저 UI 테스트 방법

---

## 1. 사전 설정

### Chrome 디버깅 모드 alias 설정

```bash
# ~/.zshrc 또는 ~/.bashrc에 추가
echo "alias chrome-debug='/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222'" >> ~/.zshrc

# 적용
source ~/.zshrc
```

### MCP 서버 설치

**Claude Code**
```bash
claude mcp add chrome-devtools -- npx chrome-devtools-mcp@latest
```

**Cursor**
```json
// ~/.cursor/mcp.json
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["chrome-devtools-mcp@latest"]
    }
  }
}
```

---

## 2. 테스트 실행 방법

### Step 1: Chrome 디버깅 모드 실행
```bash
chrome-debug
```

### Step 2: AI에게 테스트 요청

```
# 기본 형식
"http://localhost:3000/sample/list 페이지 테스트해줘"

# 상세 요청
"로그인 페이지에서 테스트용 계정 정보로 로그인 테스트"

# 검증 요청
"현재 페이지에서 콘솔 에러 있는지 확인해줘"
```

---

## 3. 자주 사용하는 테스트 패턴

### 페이지 로드 확인
```
"http://localhost:3000/sample/list 페이지 열고 정상 로드되는지 확인"
```

### 로그인 테스트
```
"로그인 페이지에서 테스트용 계정으로 로그인 버튼 클릭"
```

### 검색 기능 테스트
```
"검색창에 '테스트' 입력하고 검색 버튼 클릭 후 결과 확인"
```

### 폼 입력 테스트
```
"등록 버튼 클릭 → 모달에서 이름: 홍길동, 코드: TEST001 입력 → 저장"
```

### 에러 확인
```
"현재 페이지 콘솔 에러 확인해줘"
"API 요청 중 실패한 것 있는지 확인"
```

### 스크린샷
```
"현재 화면 스크린샷 찍어줘"
```

---

## 4. MCP 주요 명령어 (참고용)

| 명령어 | 용도 |
|--------|------|
| `navigate_page` | 페이지 이동 |
| `take_snapshot` | 화면 요소 확인 |
| `take_screenshot` | 스크린샷 저장 |
| `click` | 요소 클릭 |
| `fill` | 값 입력 |
| `fill_form` | 다중 입력 |
| `list_console_messages` | 콘솔 로그 확인 |
| `list_network_requests` | API 요청 확인 |
| `wait_for` | 텍스트 대기 |
