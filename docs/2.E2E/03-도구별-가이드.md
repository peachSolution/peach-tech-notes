# 도구별 설치·연결·명령어 가이드

각 브라우저 자동화 도구의 설치, CDP 연결, 핵심 명령어 레퍼런스.

---

## 왜 Chrome Beta를 사용하는가

기존에 사용하던 Chrome 브라우저를 그대로 유지한 채, E2E 테스트·자동화 전용 브라우저를 **별도로 실행**하여 분리한다.

- Chrome (기본) — 평소 업무·웹서핑 용도 (간섭 없음)
- Chrome Beta — E2E 테스트·AI 브라우저 자동화 전용

두 브라우저는 **완전히 독립된 앱**으로 동시 실행 가능하며, 프로필·쿠키·세션이 서로 영향을 주지 않는다.

> 설치: https://www.google.com/chrome/beta/

---

## 공통: Chrome Beta CDP 실행

모든 CDP 기반 도구 (Playwright JS API, playwright-cli, browser-use, agent-browser)가 공유하는 브라우저.

```bash
# 이미 실행 중인지 확인
curl -s http://127.0.0.1:9222/json/version

# 응답이 없으면 Chrome Beta를 CDP 모드로 실행
/Applications/Google\ Chrome\ Beta.app/Contents/MacOS/Google\ Chrome\ Beta \
  --remote-debugging-port=9222 \
  --remote-allow-origins=* \
  --user-data-dir="$HOME/.chrome-beta-e2e-profile" &

# 실행 확인 (2~3초 대기 후)
curl -s http://127.0.0.1:9222/json/version
# → { "Browser": "Chrome/147.x.x.x", ... } 출력되면 성공
```

실행 후 **사람이 브라우저에서 로그인**을 완료한다. AI는 이후 CDP로 연결하여 이어받는다.
로그인 세션은 프로필(`~/.chrome-beta-e2e-profile`)에 저장되어 다음 실행 시에도 유지된다.

> Chrome Beta 설치: https://www.google.com/chrome/beta/

## 1. Playwright JS API (CDP 연결)

### 설치

```bash
npm install playwright
```

### CDP 연결

```javascript
const { chromium } = require('playwright');
const browser = await chromium.connectOverCDP('http://127.0.0.1:9222');
const context = browser.contexts()[0];
const page = context.pages().find(p => p.url().includes('대상도메인')) || context.pages()[0];
```

### 핵심 API

```javascript
// 페이지 이동
await page.goto('https://example.com', { waitUntil: 'domcontentloaded' });

// 요소 클릭
await page.locator('a', { hasText: '링크텍스트' }).click();

// 텍스트 입력
await page.locator('#input_id').fill('입력값');

// iframe 내부 접근
const frame = page.frameLocator('iframe[src*="path"]');
await frame.locator('#element').click();

// 텍스트 추출
const text = await page.locator('.selector').innerText();

// JS 실행
const result = await page.evaluate(() => document.title);

// 대기
await page.waitForSelector('tr.zA', { timeout: 10000 });
await page.waitForURL('**/expected/**', { timeout: 15000 });
```

### codegen (녹화 → 코드 자동 생성)

```bash
# 세션 저장 (CDP 연결에서 쿠키 추출)
node -e "
const { chromium } = require('playwright');
(async () => {
  const browser = await chromium.connectOverCDP('http://127.0.0.1:9222');
  await browser.contexts()[0].storageState({ path: 'auth.json' });
  await browser.close();
})();
"

# 저장된 세션으로 녹화 시작
npx playwright codegen --load-storage=auth.json -o recorded.js 'https://example.com'

# CDP 연결된 브라우저에서 Inspector 열기 (codegen 유사)
# → JS API에서 page.pause() 호출
```

### iframe/모달 처리 패턴

```javascript
// jQuery UI 모달 내 iframe 접근
await page.locator('span.button a', { hasText: '버튼명' }).click();
const frame = page.frameLocator('iframe[src*="target/path"]');
await frame.locator('#list').waitFor({ timeout: 8000 });
await frame.locator('#keyword').fill('검색어');
await frame.locator('input[type="submit"]').click();
```

## 2. playwright-cli — AI 에이전트 전용 ([GitHub](https://github.com/microsoft/playwright-cli))

> `npx playwright`(본체 CLI)와 **완전히 다른 별도 패키지**.
> AI 코딩 에이전트가 스크립트 없이 브라우저를 조작하기 위한 도구.

### 설치

```bash
npm install -g @playwright/cli@latest
```

### 두 가지 운용 방식

playwright-cli는 **CDP 연결 방식** (권장)과 **자체 브라우저 방식** 두 가지로 운용할 수 있다.

| 방식 | 브라우저 | 장점 | 단점 |
|------|---------|------|------|
| **CDP 연결 (권장)** | 기존 Chrome Beta (CDP 9222) | Playwright JS API와 **동일 브라우저 공유**, 로그인 1회 | 설정 파일 필요 |
| 자체 브라우저 | playwright-cli 전용 Chrome Beta | 설정 없이 바로 사용 | 별도 브라우저, 별도 로그인 필요 |

---

### 방식 1: CDP 연결 (권장)

Playwright JS API와 **동일한 Chrome Beta 1개**를 공유하는 방식.
사람이 1회 로그인하면 playwright-cli와 E2E 스크립트 양쪽에서 모두 사용 가능.

#### CDP 설정 파일

`~/.playwright/cli.config.json` (전역 설정):

```json
{
  "browser": {
    "cdpEndpoint": "http://localhost:9222",
    "isolated": false
  }
}
```

> **`isolated: false` 필수** — 기본값이 `true`라서, CDP 연결해도 새 브라우저가 열린다.
> ([GitHub #270](https://github.com/microsoft/playwright-cli/issues/270)에서 Playwright 팀이 직접 확인)

#### shell alias 설정

매번 `--config` 옵션을 타이핑하지 않으려면 `~/.zshrc`에 alias를 추가한다:

```bash
# ~/.zshrc
alias pwc='playwright-cli --config=$HOME/.playwright/cli.config.json'
```

#### CDP 연결 사용법

```bash
# 1. Chrome Beta CDP 실행 (위 "공통: Chrome Beta CDP 실행" 참조)
# 이미 실행 중인지 확인: curl -s http://127.0.0.1:9222/json/version

# 2. 브라우저에서 사람이 직접 로그인

# 3. playwright-cli CDP 연결
playwright-cli --config=$HOME/.playwright/cli.config.json open
# 또는 alias 사용
pwc open

# 4. 기존 탭 확인
pwc tab-list
# → 0: [NAVER](https://www.naver.com/)
# → 1: [개발 사이트](https://dev.example.com/)
# → 2: [Gmail](https://mail.google.com/...)

# 5. AI가 이어서 사용
pwc snapshot
pwc click e11
pwc eval "document.title"
```

#### 통합 구성도

```
Chrome Beta (CDP 9222) — 사람이 1회 로그인
    │
    ├── playwright-cli (--config=~/.playwright/cli.config.json)
    │   → AI 브라우저 탐색 (snapshot, click, eval)
    │
    └── node 시나리오/*.js (connectOverCDP)
        → E2E 반복 시나리오
```

> **`pwc close`** 해도 Chrome Beta는 살아있다 (CDP 연결만 끊김).
> 다시 `pwc open` 하면 즉시 재연결.

### 방식 2: 자체 브라우저 (독립 운용)

설정 파일 없이 playwright-cli가 자체 Chrome Beta를 실행하는 방식.
CDP Chrome Beta와 **별도 프로필**을 사용한다.

#### 기본 운용 지침

1. **Chrome Beta를 사용**한다 (`--browser=chrome-beta`)
2. **사람이 처음 한 번 로그인**한다 (Google, 대상 서비스 등)
3. **브라우저를 닫지 않는다** (`close` 하지 않음) — AI가 언제든 이어서 사용
4. Claude Code 세션이 끝나도 브라우저는 살아있음 (별도 프로세스)

#### 최초 실행 (사람이 1회만)

```bash
# Chrome Beta + headed + persistent로 열기
playwright-cli open "https://accounts.google.com" --browser=chrome-beta --headed --persistent

# → 브라우저에서 사람이 직접 로그인
# → close 하지 않고 그대로 둠
```

#### AI가 이어서 사용

```bash
# 세션 상태 확인
playwright-cli list
# → status: open  ← 이 상태면 바로 명령 가능

# 브라우저가 닫혀있다면 재실행 (로그인 유지됨)
playwright-cli open "https://www.google.com" --browser=chrome-beta --headed --persistent
```

#### 프로필 경로

| 항목 | 경로 |
|------|------|
| **자동 프로필** | `~/Library/Caches/ms-playwright/daemon/.../ud-default-chrome-beta` |
| **지정 프로필** | `--profile=/원하는/경로` |
| **CDP Chrome Beta 프로필** | `~/.chrome-beta-e2e-profile` (별도, 동시 사용 불가) |

> 자체 브라우저 방식은 CDP Chrome Beta와 **프로필을 공유할 수 없다** (잠금 충돌).
> 각각 독립 프로필로 운용하고, 사람이 **각각 한 번씩만 로그인**한다.

### 핵심 명령어

```bash
# 페이지 이동
playwright-cli goto "https://mail.google.com"

# 페이지 스냅샷 (ref 목록 포함, .playwright-cli/*.yml에 저장)
playwright-cli snapshot

# ref로 클릭
playwright-cli click e10

# ref에 텍스트 입력
playwright-cli fill e37 "검색어"

# 키보드 입력
playwright-cli press Enter

# JavaScript 실행 (단순 표현식만 — IIFE 사용 금지)
playwright-cli eval "document.title"
playwright-cli eval "document.querySelector('tr.zA .bog') ? document.querySelector('tr.zA .bog').innerText.trim() : '없음'"

# 스크린샷
playwright-cli screenshot

# 탭 관리
playwright-cli tab-list
playwright-cli tab-new "https://example.com"
playwright-cli tab-select 1

# 세션 확인
playwright-cli list
```

### 스냅샷 출력 예시

```yaml
- link "Gmail" [ref=e10] [cursor=pointer]:
    - /url: https://mail.google.com/mail/&ogbl
- button "Google 계정: 박동재" [ref=e26] [cursor=pointer]:
- combobox "검색" [active] [ref=e37]
- button "Google 검색" [ref=e64] [cursor=pointer]
```

> **snapshot 결과를 터미널에서 바로 grep하면 못 찾는 경우가 있다.**
> snapshot은 `.playwright-cli/page-*.yml` 파일에 저장되므로, yml 파일을 grep한다:
> ```bash
> grep -i "검색키워드" .playwright-cli/page-*.yml | grep "ref=" | tail -5
> ```

### browser-use CLI / agent-browser와의 차이

| 항목 | playwright-cli | browser-use CLI | agent-browser CLI |
|------|---------------|-----------------|-------------------|
| **브라우저** | **CDP 연결** 또는 자체 Chrome Beta | Chrome Beta (CDP 연결) | Chrome Beta (CDP 연결) |
| **CDP 연결** | **✅ 설정 파일 (`cdpEndpoint`)** | ✅ --cdp-url | ✅ 환경변수 |
| **세션 유지** | **CDP** 또는 --persistent (영구) | CDP 의존 | CDP 의존 |
| **close 없이 유지** | ✅ CDP 브라우저 또는 별도 프로세스 | CDP 브라우저 의존 | CDP 브라우저 의존 |
| **요소 참조** | ref (스냅샷) | element ID (state) | @eN ref (snapshot) |
| **JavaScript** | eval 지원 | eval 지원 | 미지원 |
| **제작사** | Microsoft (공식) | browser-use | Vercel Labs |

### 주의사항

- CDP 연결 시 설정 파일에 **`isolated: false` 필수** (기본값 `true`면 새 브라우저가 열림)
- **CDP는 사용자 포커스 탭을 알 수 없다** — `(current)` 표시는 playwright-cli 내부 상태이지 Chrome UI 포커스가 아님. `tab-list`로 목록을 확인하고 `tab-select N`으로 명시 지정
- **`### Page` URL/Title은 무시** — 내부 page 객체 고정값, 실제 탭과 불일치
- **eval은 단순 표현식만** — IIFE `(function(){...})()` 사용 시 `Passed function is not well-serializable!` 오류 발생. 여러 동작은 별도 eval로 분리
- **snapshot은 yml 파일 grep** — 터미널 직접 grep이 안 될 때 `.playwright-cli/page-*.yml` 파일을 grep
- 자체 브라우저 방식은 CDP Chrome Beta 프로필과 동시 사용 불가 (잠금 충돌)
- 브라우저를 사람이 수동으로 닫으면 `playwright-cli list`에서 `status: closed`가 됨 → 재실행 필요
- `--config` 옵션은 CLI 플래그로만 지정 가능 (홈 디렉토리 자동 탐색 없음)

## 3. browser-use CLI (CDP 연결)

### 설치

```bash
pip install browser-use
browser-use install
```

### CDP 연결

```bash
# 모든 명령에 --cdp-url 옵션 추가
browser-use --cdp-url http://127.0.0.1:9222 <명령>
```

### 핵심 명령어

```bash
# 페이지 이동
browser-use --cdp-url http://127.0.0.1:9222 open "https://example.com"

# DOM 트리 출력 (element ID 확인)
browser-use --cdp-url http://127.0.0.1:9222 state

# element ID로 클릭
browser-use --cdp-url http://127.0.0.1:9222 click 105

# element ID에 텍스트 입력
browser-use --cdp-url http://127.0.0.1:9222 input 99 "검색어"

# JavaScript 실행 (iframe 내부 접근 가능)
browser-use --cdp-url http://127.0.0.1:9222 eval 'document.title'

# 탭 전환
browser-use --cdp-url http://127.0.0.1:9222 switch 1

# 스크린샷
browser-use --cdp-url http://127.0.0.1:9222 screenshot /tmp/result.png
```

### element ID 찾는 법

```bash
# state에서 키워드 검색
browser-use --cdp-url http://127.0.0.1:9222 state | grep -B1 -A1 "Gmail"
# → [57]<a /> Gmail  ← 57이 element ID
```

### iframe 접근 (eval 사용)

```bash
# state로 iframe 내부 ID가 보이지만 input이 안 먹을 때 → eval로 직접 조작
browser-use --cdp-url http://127.0.0.1:9222 eval '
  var iframeDoc = document.querySelector("iframe#ui-id-4").contentDocument;
  iframeDoc.querySelector("#page_keyword").value = "검색어";
  iframeDoc.querySelector("input[type=submit]").click();
'
```

### 주의사항

- element ID는 **페이지 로딩마다 변경** → 반드시 `state`로 매번 확인
- `input` 명령은 iframe shadow DOM에 값 전달이 안 될 수 있음 → `eval` 사용
- `--cdp-url`을 매 명령마다 붙여야 함

## 4. agent-browser CLI (CDP 연결)

### 설치

```bash
npm i -g agent-browser
# 또는
brew install agent-browser
```

### CDP 연결

```bash
# 환경변수로 CDP 주소 지정
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser <명령>
```

### 핵심 명령어

```bash
# 페이지 이동
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser open "https://example.com"

# 접근성 트리 스냅샷 (인터랙티브 요소만, 컴팩트)
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser snapshot -i -c

# ref로 클릭
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser click e9

# ref에 텍스트 입력
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser fill e5 "검색어"

# 키보드 입력
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser press e5 Enter

# 텍스트 추출
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser get text e1

# 스크린샷 (로컬 저장 → 토큰 미소비)
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser screenshot --path result.png
```

### snapshot 옵션

| 옵션 | 설명 |
|------|------|
| `-i` | 인터랙티브 요소만 (버튼, 링크, 입력창) |
| `-c` | 컴팩트 출력 |
| `-d N` | DOM 깊이 제한 |
| `-s "selector"` | CSS 범위 제한 |

### 한계

- **iframe 내부 접근 불가**: snapshot에 `iframe` 태그는 보이지만 내부 요소 조작 불가
- jQuery UI 모달 + iframe 구조에서 사용 불가
- iframe이 없는 단순 페이지에서만 유효

## 5. claude-in-chrome MCP

### 사전 준비

- Chrome 웹스토어에서 "Claude for Chrome" 설치 (v1.0.64+)
- Claude Code CLI와 동일한 claude.ai 계정 로그인
- `/mcp list` → `claude-in-chrome ✔ connected` 확인

### 워크플로우

```
1. ToolSearch: select:mcp__claude-in-chrome__tabs_context_mcp
2. tabs_context_mcp(createIfEmpty: true) → tabId 획득
3. navigate(tabId, url)                  → 페이지 이동
4. find(tabId, "자연어 쿼리")            → 요소 ref 획득
5. computer(tabId, left_click, ref)      → 클릭
6. get_page_text(tabId)                  → 텍스트 추출
7. javascript_tool(tabId, "JS코드")      → JS 실행
8. computer(tabId, screenshot)           → 스크린샷
```

### 주요 도구

| 도구 | 용도 |
|------|------|
| `tabs_context_mcp` | 탭 목록 조회 (항상 첫 번째) |
| `navigate` | URL 이동 |
| `find` | 자연어로 요소 탐색 |
| `computer` | 스크린샷, 클릭, 타이핑 |
| `get_page_text` | 페이지 텍스트 추출 |
| `javascript_tool` | JS 직접 실행 |
| `read_console_messages` | 콘솔 로그 |
| `read_network_requests` | 네트워크 모니터링 |

### 토큰 절약 패턴

- screenshot 최소화 (클릭 전 1회, 검증 시 1회)
- 텍스트는 `get_page_text` 또는 `javascript_tool` 우선
- `find` 도구로 요소 ref 획득 후 클릭 (좌표 추정 불필요)
