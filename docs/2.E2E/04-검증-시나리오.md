# 검증 시나리오: 재현 가능한 코드

실제 검증에 사용한 시나리오와 재현 가능한 코드. 새로운 도구 도입 시 동일 시나리오로 비교 가능.

---

## 사전 준비 (공통)

> **Chrome Beta를 사용하는 이유**: 기존 Chrome 브라우저는 평소대로 유지하고,
> E2E·자동화 전용 브라우저를 별도로 분리하여 서로 간섭 없이 사용한다.

### 1단계: Chrome Beta 설치 (최초 1회)

Chrome Beta 다운로드: https://www.google.com/chrome/beta/

macOS에서 기본 Chrome과 Chrome Beta는 **완전히 별개의 앱**으로 동시 실행 가능.

### 2단계: Chrome Beta CDP 모드 실행

```bash
# 이미 실행 중인지 확인
curl -s http://127.0.0.1:9222/json/version

# 응답이 없으면 Chrome Beta를 CDP 모드로 실행
/Applications/Google\ Chrome\ Beta.app/Contents/MacOS/Google\ Chrome\ Beta \
  --remote-debugging-port=9222 \
  --remote-allow-origins=* \
  --user-data-dir="$HOME/.chrome-beta-e2e-profile" &

# 실행 확인 (2~3초 대기 후)
curl -s http://127.0.0.1:9222/json/version
# → { "Browser": "Chrome/147.x.x.x", ... } 출력되면 성공
```

### 3단계: 브라우저에서 로그인

Chrome Beta 창이 열리면 **사람이 직접 로그인**한다 (Google, 대상 서비스 등).
로그인 세션은 프로필에 저장되어 다음 실행 시에도 유지된다.

### 4단계: 도구별 연결

**Playwright JS API / browser-use / agent-browser** — 추가 설정 없이 CDP 연결:

```bash
# JS API 예시
node -e "const {chromium}=require('playwright'); chromium.connectOverCDP('http://127.0.0.1:9222').then(b=>console.log('연결됨'))"
```

**playwright-cli (CDP 연결 — 권장)** — 설정 파일 필요:

```bash
# 최초 1회: 설정 파일 생성
mkdir -p ~/.playwright
cat > ~/.playwright/cli.config.json << 'EOF'
{
  "browser": {
    "cdpEndpoint": "http://localhost:9222",
    "isolated": false
  }
}
EOF

# 최초 1회: shell alias 추가 (선택)
echo "alias pwc='playwright-cli --config=\$HOME/.playwright/cli.config.json'" >> ~/.zshrc

# CDP 연결
playwright-cli --config=$HOME/.playwright/cli.config.json open
# 또는 alias: pwc open

# 기존 탭 확인 — CDP Chrome Beta의 탭이 그대로 보임
pwc tab-list
```

> **`isolated: false` 필수** — 없으면 CDP에 연결해도 새 브라우저가 열린다.

**playwright-cli (자체 브라우저 — 대안)** — CDP 없이 독립 운용:

```bash
# Chrome Beta + persistent로 열기 (최초 1회)
playwright-cli open "https://accounts.google.com" --browser=chrome-beta --headed --persistent

# 브라우저에서 사람이 직접 로그인 → close 하지 않음
playwright-cli list  # → status: open 확인
```

## 시나리오 1: Gmail 첫 번째 메일 제목

**목적**: google.com → Gmail 링크 찾아 클릭 → 받은편지함 → 첫 번째 메일 제목 추출
**검증 포인트**: 페이지 이동, 링크 탐색·클릭, 텍스트 추출

### Playwright JS API

```javascript
const { chromium } = require('playwright');
(async () => {
  const t0 = Date.now();
  const browser = await chromium.connectOverCDP('http://127.0.0.1:9222');
  const context = browser.contexts()[0];
  let page = context.pages().find(p => p.url().includes('google.com')) || context.pages()[0];

  // google.com 이동
  await page.goto('https://www.google.com', { waitUntil: 'domcontentloaded' });

  // Gmail 링크 클릭
  await page.locator('a[href*="mail.google.com"], a:has-text("Gmail")').first().click();
  await page.waitForURL('**/mail.google.com/**', { timeout: 15000 });
  await page.waitForSelector('tr.zA', { timeout: 15000 });

  // 첫 번째 메일 제목
  const subject = await page.locator('tr.zA .bog, tr.zA .y6 span').first().innerText();
  console.log(`첫번째 메일: "${subject}"`);
  console.log(`소요: ${Date.now()-t0}ms`);

  await browser.close();
  process.exit(0);
})();
// 실측 결과: 3,773ms, 토큰 0
```

### browser-use CLI

```bash
START=$(date +%s%N | cut -b1-13)

# google.com 이동
browser-use --cdp-url http://127.0.0.1:9222 open "https://www.google.com"

# Gmail 링크 ID 탐색
GMAIL_ID=$(browser-use --cdp-url http://127.0.0.1:9222 state \
  | grep -i "gmail" | grep -o '\[[0-9]*\]' | head -1 | tr -d '[]')

# Gmail 클릭
browser-use --cdp-url http://127.0.0.1:9222 click $GMAIL_ID
sleep 2

# 첫 번째 메일 제목 추출
browser-use --cdp-url http://127.0.0.1:9222 eval \
  'document.querySelector("tr.zA .bog")?.textContent?.trim()'

END=$(date +%s%N | cut -b1-13)
echo "소요: $((END-START))ms"
# 실측 결과: 13,273ms, 토큰 ~800
```

### agent-browser CLI

```bash
START=$(date +%s%N | cut -b1-13)

# google.com 이동
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser open "https://www.google.com"

# Gmail 링크 ref 찾기
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser snapshot -i -c | grep Gmail
# → Gmail [ref=e3]

# Gmail 클릭
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser click e3

# 메일 목록 확인 (접근성 트리에서 제목 추출)
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser snapshot -i -c | head -20

END=$(date +%s%N | cut -b1-13)
echo "소요: $((END-START))ms"
# 실측 결과: ~5,300ms, 토큰 ~1,400
```

### playwright-cli (CDP 연결 방식)

```bash
# 사전: Chrome Beta CDP 실행 중 (위 "사전 준비" 2단계 참조)
# 사전: ~/.playwright/cli.config.json 설정 완료
# 사전: alias pwc='playwright-cli --config=$HOME/.playwright/cli.config.json'

# CDP 연결 (이미 open 상태면 생략)
pwc open

START=$(date +%s%N | cut -b1-13)

# google.com 이동
pwc goto "https://www.google.com"

# 스냅샷에서 Gmail ref 확인
pwc snapshot
# → .playwright-cli/*.yml 에서 link "Gmail" [ref=e11] 확인

# Gmail 클릭
pwc click e11

# eval로 첫 번째 메일 제목 추출
pwc eval "document.querySelector('tr.zA .bog') ? document.querySelector('tr.zA .bog').innerText.trim() : '없음'"

# ※ pwc close 해도 Chrome Beta는 살아있음 (CDP 연결만 끊김)

# 실측 결과: ~3,900ms, 토큰 낮음 (스냅샷 텍스트)
# 특징: CDP Chrome Beta와 동일 브라우저 공유, E2E JS API와 병행 가능
```

## 시나리오 2: 처방명 검색 (jQuery UI 모달 + iframe)

**목적**: 상세 입력 페이지 → 검색 버튼 → 모달/iframe → 키워드 검색 → 항목 선택 → 모달 닫힘
**검증 포인트**: 모달 처리, iframe 내부 접근, 폼 입력, 버튼 클릭

### Playwright JS API

```javascript
const { chromium } = require('playwright');
(async () => {
  const t0 = Date.now();
  const browser = await chromium.connectOverCDP('http://127.0.0.1:9222');
  const context = browser.contexts()[0];
  let page = context.pages().find(p => p.url().includes('dev.example.com')) || context.pages()[0];

  // 상세 입력 페이지 이동
  const origin = new URL(page.url()).origin;
  await page.goto(`${origin}/sample/form/step1`);
  await page.waitForLoadState('networkidle');

  // 검색 버튼 클릭 → 모달 오픈
  await page.locator('span.button a', { hasText: '항목검색' }).click();

  // iframe 로드 대기
  const frame = page.frameLocator('iframe[src*="formulary/search"]');
  await frame.locator('#list_inner_html').waitFor({ timeout: 8000 });

  // 키워드 검색
  await frame.locator('#page_keyword').fill('테스트항목');
  await frame.locator('input[type="submit"]').click();
  await page.waitForTimeout(800);

  // 정확히 '테스트항목'인 첫 번째 행의 선택 클릭
  const rows = frame.locator('#list_inner_html tr');
  const count = await rows.count();
  for (let i = 0; i < count; i++) {
    const td = await rows.nth(i).locator('td').first().innerText().catch(() => '');
    if (td.trim() === '테스트항목') {
      await rows.nth(i).locator('a', { hasText: '선택' }).click();
      break;
    }
  }

  // 추가 선택 다이얼로그 처리 (있을 경우)
  await page.waitForTimeout(800);
  const hapBtn = page.locator('button', { hasText: '추가선택' });
  if (await hapBtn.isVisible({ timeout: 1500 }).catch(() => false)) {
    await hapBtn.click();
  }

  // 선택 결과 확인
  const prescName = await page.locator('#selected_item_name').inputValue();
  console.log(`선택 항목: "${prescName}"`);
  console.log(`소요: ${Date.now()-t0}ms`);

  await browser.close();
  process.exit(0);
})();
// 실측 결과: 7,531ms, 토큰 0, headed 모드에서 눈으로 확인 가능
```

### playwright-cli (CDP 연결 + eval 단순 표현식 방식)

```bash
# 사전: Chrome Beta CDP 실행 + pwc open 완료
pwc list  # → status: open 확인

# 상세 입력 페이지 이동
pwc goto "https://dev.example.com/sample/form/step1"

# 검색 버튼 ref 확인
pwc snapshot
# → .playwright-cli/page-*.yml 에서 link "항목검색" [ref=e94] 확인

# 검색 버튼 클릭 → 모달/iframe 오픈
pwc click e94

# iframe 내부에서 테스트항목 검색 — 별도 eval로 분리 (IIFE 불가)
pwc eval "document.querySelector('iframe[src*=formulary]').contentDocument.querySelector('#page_keyword').value = '테스트항목'"
pwc eval "document.querySelector('iframe[src*=formulary]').contentDocument.querySelector('input[type=submit]').click()"
sleep 1

# 검색 결과 확인
pwc eval "document.querySelector('iframe[src*=formulary]').contentDocument.querySelector('#list_inner_html').innerText.substring(0,300)"

# 첫 번째 행의 선택 링크 클릭
pwc eval "document.querySelector('iframe[src*=formulary]').contentDocument.querySelector('#list_inner_html tr a').click()"

# 선택 결과 확인
pwc eval "document.querySelector('#selected_item_name').value"

# ※ pwc close 해도 Chrome Beta는 살아있음
# 실측 결과: ~6,800ms, 토큰 낮음
# 핵심: CDP Chrome Beta와 동일 브라우저 + snapshot ref + eval 단순 표현식으로 iframe 조작
# 주의: eval은 단순 표현식만 가능. IIFE (function(){...})() 는 직렬화 오류 발생
```

### browser-use CLI (eval 방식)

```bash
# 검색 모달이 열린 상태에서 iframe 접근
# ※ state로 보이는 ID에 input을 해도 iframe shadow DOM 문제로 실패
# ※ eval로 직접 접근해야 함

# 1. 모든 iframe 확인
browser-use --cdp-url http://127.0.0.1:9222 eval \
  'Array.from(document.querySelectorAll("iframe")).map(f => f.id + ": " + f.src)'

# 2. 올바른 iframe에 검색어 입력 + 검색 실행
browser-use --cdp-url http://127.0.0.1:9222 eval '
  var iframeDoc = document.querySelector("iframe#ui-id-4").contentDocument;
  iframeDoc.querySelector("#page_keyword").value = "테스트항목";
  iframeDoc.querySelector("input[type=submit]").click();
'

# 3. 검색 결과 확인
sleep 1.5
browser-use --cdp-url http://127.0.0.1:9222 eval \
  'document.querySelector("iframe#ui-id-4").contentDocument.querySelector("#list_inner_html").innerText.substring(0,300)'

# 4. 테스트항목 선택 클릭
browser-use --cdp-url http://127.0.0.1:9222 eval '
  var iframeDoc = document.querySelector("iframe#ui-id-4").contentDocument;
  var rows = Array.from(iframeDoc.querySelectorAll("#list_inner_html tr"));
  var target = rows.find(r => r.querySelector("td") && r.querySelector("td").textContent.trim() === "테스트항목");
  if(target) target.querySelector("a").click();
'

# 실측 결과: ~18,000ms, iframe ID 탐색에 시행착오 발생
# 주의: iframe ID(ui-id-2, ui-id-4)는 모달 열기 횟수에 따라 변경됨
```

### agent-browser CLI → 실패

```bash
# snapshot에 iframe 태그는 보이나 내부 요소 접근 불가
AGENT_BROWSER_CDP=http://127.0.0.1:9222 agent-browser snapshot 2>&1 | grep iframe
# → iframe  (내부 요소 없음)

# ❌ jQuery UI 모달 + iframe 구조에서 agent-browser 사용 불가
```

## 검증 시 주의사항

1. **Gmail 셀렉터**: `tr.zA .bog` — Gmail UI 업데이트로 변경 가능
2. **iframe ID 변동**: jQuery UI 모달은 열 때마다 iframe ID가 증가 (ui-id-2, ui-id-4, ...)
3. **networkidle 타임아웃**: Gmail 같은 SPA는 `domcontentloaded` 사용 권장
4. **합방 다이얼로그**: 처방명 투여 시 기존 처방이 있으면 합방/대체 선택 다이얼로그 출현
5. **CDP 탭 포커스**: CDP는 사용자가 보고 있는 탭을 알 수 없음. `tab-list` → 탭 번호 확인 → `tab-select`로 명시 지정
6. **playwright-cli eval**: IIFE `(function(){...})()` 불가. 단순 표현식만 사용, 여러 동작은 별도 eval로 분리
7. **snapshot grep**: 터미널에서 바로 grep 안 될 때 `.playwright-cli/page-*.yml` 파일을 grep
