# Git 워크플로우 가이드

> 팀 표준 Git 워크플로우

---

## 🌳 브랜치 구조

```
main     ← 운영 배포
  ↑
dev      ← 개발 서버 배포
  ↑
develop  ← 개발 통합
  ↑
feature/ ← 기능 개발
```

---

## 🎯 핵심 규칙 3가지

### 1️⃣ 작업 브랜치 동기화 → **Rebase**

```bash
# ❌ 금지
git merge develop

# ✅ 사용
git fetch origin
git rebase origin/develop
git push --force-with-lease
```

### 2️⃣ PR 머지 → **Merge Commit (기본)**

| 방향 | 머지 전략 | GitHub 버튼 |
|------|----------|-------------|
| develop → dev | **Merge Commit** | Create a merge commit |
| dev → main | **Merge Commit** | Create a merge commit |

### 3️⃣ 작업 브랜치 → develop → **Squash Merge (선택)**

| 방향 | 머지 전략 | GitHub 버튼 |
|------|----------|-------------|
| feature → develop | Squash Merge | Squash and merge |

> 💡 커밋이 많거나 정리가 필요할 때 Squash 적용, 그 외엔 Merge Commit도 가능

---

## ⚠️ 왜 이렇게 해야 하나?

### Squash Merge를 develop → dev에 사용하면 충돌 반복 발생

```
실제 발생한 문제:

1. PR #24: feature → develop (파일 A 생성)
2. PR #25: develop → dev (Squash Merge) → 새 커밋 해시 생성
3. develop에서 파일 A 수정
4. PR #26: develop → dev → 💥 충돌!

원인: Squash Merge는 커밋 히스토리 연결을 끊음
     → Git이 "같은 파일을 두 곳에서 수정"으로 인식
```

**Git 공식 권장:**
> "For repeated merges of long-lived branches, using regular merge commits is recommended."

## 📋 요약표

| 상황 | 전략 | 비고 |
|------|------|------|
| feature에서 develop 최신화 | **Rebase** | 선형 이력 유지 |
| feature → develop PR | Squash Merge | 선택적 적용 |
| develop → dev PR | **Merge Commit** | 필수 |
| dev → main PR | **Merge Commit** | 필수 |
| 공유 브랜치 (develop, dev, main) | Rebase 금지 | 히스토리 보호 |

## 🔄 작업 흐름

### 1. 기능 개발 시작

```bash
git checkout develop
git pull origin develop
git checkout -b feature/기능명
```

### 2. 작업 중 동기화 (필요시)

```bash
git fetch origin
git rebase origin/develop
git push --force-with-lease
```

### 3. PR 생성 및 머지

- **feature → develop**: Squash and merge (선택)
- **develop → dev**: ⚠️ **Create a merge commit** (필수)
- **dev → main**: ⚠️ **Create a merge commit** (필수)

### 4. 작업 완료 후 정리

```bash
git checkout develop
git pull origin develop
git branch -d feature/기능명
git fetch --prune
```

## ⚙️ 초기 설정

### Git 설정 (전체 팀원)

```bash
git config --global pull.rebase true
```

### GitHub 저장소 설정 (관리자)

```
Settings → General → Pull Requests
```

| 설정 | 상태 |
|------|------|
| Allow merge commits | ☑️ 활성화 |
| Allow squash merging | ☑️ 활성화 |
| Automatically delete head branches | ☑️ 활성화 |

## 🔧 트러블슈팅

### Rebase 충돌 해결

```bash
# 1. 충돌 파일 수정
# 2. 계속 진행
git add .
git rebase --continue

# 취소하려면
git rebase --abort
```

### Force Push 안전하게

```bash
# ❌ 위험
git push --force

# ✅ 안전
git push --force-with-lease
```

## 📚 참고 명령어

```bash
# 브랜치
git checkout -b feature/name      # 생성
git branch -d feature/name        # 삭제
git branch -a                     # 조회

# 동기화
git fetch origin                  # 원격 정보 가져오기
git rebase origin/develop         # Rebase
git pull origin develop           # Pull (rebase 설정 시)

# Push
git push origin feature/name                     # 일반
git push origin feature/name --force-with-lease  # Force

# Rebase
git rebase --continue             # 계속
git rebase --abort                # 취소

# 정리
git fetch --prune                 # 원격 브랜치 정리
```
